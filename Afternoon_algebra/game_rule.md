## 《六色弹珠：六边形碰撞战》—— 网络游戏规则书v1.2

### 一、游戏目标
在10×10×10的六边形棋盘上，利用**弹性碰撞**与**棋子特性**，将对方所有弹珠击出棋盘（或使其死亡），即获得胜利。

### 二、棋盘与坐标系统
- **棋盘形状**：正六边形区域，每条边包含8个格点。总格点数 = 1 + 6× (1+2+…+7) = 169个。
- **坐标系统**：取中心点位为坐标原点，两条夹角为120°的直线作为x，y坐标轴。取x,y均为整数的点作为格点。则原点周围的六个点为：(1,0) (1,1) (0,1) (-1,0) (-1,-1) (0,-1)
- **边界判定**：考虑$D = \max\{ |x|, |y|, |x-y| \}$ ,如果$D \ge 8$，则出界。

### 三、初始配置
1. **双方阵营**：红方（左方） / 蓝方（右方）。
2. **初始位置**：棋盘的两端（红方占据 x∈[-7，-2] 且 $D \le 8$ 的角落区；蓝方占据 x∈[2,7] 且 $D \le 8$  的角落区）。具体坐标通过手动选择每个弹珠的位置。
3. **选珠阶段**：
   - 每方拥有 **6个弹珠**。
   - 从6种颜色中**任意选取**（可重复，例如6个全是黄球）。
   - 选珠完成后，双方自行按照先后手对弹珠的阵型进行手动的排布。
4.  **先后手**：随机决定，红方或蓝方先行动。

### 四、回合流程（网络同步）
每个回合按以下顺序执行：

1. **玩家选择**一个己方存活弹珠。
2. **根据弹珠颜色，执行其专属移动指令**（详见第五节）。
3. **结算所有碰撞、出界、死亡、特性触发**（按物理顺序，见第六条）。
4. **检查胜负**：若一方弹珠数量为0，游戏结束。
5. **切换回合**，对方行动。

> 注：所有随机性结果（如力度偏移、黑球方向、随从生成位置）由**服务器**生成并广播，保证双方一致。

### 五、六种弹珠的详细规则（随机性全部保留）

#### 1. 白球（基础）
- **移动方式**：指定方向（6选1） + 指定基础力度（1~5）。  
  实际移动步数 = 基础力度。
- 当白球被友方弹珠碰撞时，在第六条的碰撞逻辑基础之上，白球额外获得+1剩余步数。如，若蓝球碰到白球时剩余步数为4，则白球向原本方向移动五格。
- **特殊能力**：每当己方**其他颜色**弹珠死亡时（黄球除外，见白球说明），若白球仍存活，则**立即**变为该死亡弹珠的颜色，并获得其全部特性（包括已累积的增益）。  
  - 若同一时刻多颗不同颜色死亡（例如碰撞同时出界），白球随机选择其中一种颜色变化。  <font color='red'>每次有己方弹珠死亡只会变化一个己方白球.</font>
  - 白球变化后的颜色**可以再次变化**（后续其他颜色死亡时再次覆盖）。 <font color='red'>若场上己方同时存在多个白球且尚有白球未变色，则优先将未变色的白球变色。若场上已无未变色白球，则已变色的白球可以被‘覆盖’为更新死亡的弹珠颜色，实现战术调整。</font>
  - 白球**不会**因黄球死亡而变色（黄球死亡只触发增益，不视为“其他颜色弹珠”用于白球变色）。

#### 2. 蓝球（随从）

- **选择移动方向**：  
    蓝球先选择一个移动方向（共 6 个可能方向，与六边形网格相邻方向一致）。

- **生成随从**（在移动前）：  
    在与所选移动方向**不共线**的相邻 4 个格点中，选择**未被占据**的格点生成随从。  
    优先选择位于移动路线**两侧各一格**（即与移动方向夹角为 ±60∘±60∘ 的两个位置）。  
    若某个优先位置被占据，则从剩余的不共线邻格中随机挑选空闲格补足。  
    最多生成 **2 个随从**。若可选空闲格少于 2 个，则能生成几个就生成几个（可能 0 个）。

- **蓝球移动**：  
    蓝球本身按照与白球相同的移动规则（方向 + 力度 11 ∼ 55）移动。

- **随从移动**：  
    两个随从（若存在）以**完全相同的方向和力度**进行移动，独立结算碰撞。

- **随从消失**：  
    随从移动结束后立即消失，无论是否存活或碰撞。

- **蓝球死亡条件**（重要）：
    
    - 蓝球自身出界 → 蓝球死亡。
    
    - **任意一个随从出界** → 蓝球死亡（即使蓝球本身未出界）。
    
    - 随从与敌方弹珠碰撞**不会**导致蓝球死亡，仅正常交换速度。
    
    - 若蓝球在移动前生成随从时周围无空闲格点（0 个随从），则蓝球移动后不会因“随从出界”而死亡。

#### 3. 绿球（推挤）
- **移动方式**：与白球相同（方向 + 力度1~5 ）。  
- **移动后特性**：绿球停止后，检查其**相邻6个格点**。  
  - 对每个相邻格上的弹珠（不分敌我），将其**沿远离绿球的方向**推开1格。  ==进行判定机制的完善==
  - **明确规则**：若目标格被占据，则推开失败，该弹珠停留在原处且不死亡。若目标格出界，该弹 珠死亡。
  - 所有推开**同时结算**（即不会因A推开后占用了B的目标格而影响B）。
  ==optional:以下内容可以进行讨论或在第二周实现==
  -  允许**链式推挤**：若目标格被占，则继续推那个球（递归一次）
  -  但需要限制深度（最多1次），避免无限推

#### 4. 红球（定向步进）
- **移动方式**：不指定力度，改为指定 **移动步数 L（1~4）**，然后**逐步为每一步选择一个方向**（每步可从6个方向中任选，允许折返）。（相当于连续进行L次力度为1的运动，每一步可以自选方向。）
- **碰撞处理**：每一步移动后**立即**与所在格点上的弹珠进行弹性碰撞（速度交换）。  
  - 若某一步碰撞后导致自己或对方出界，则出界者立即死亡并**终止后续移动**。  
- **与随从/推挤的交互**：红球移动过程中不会触发绿球的推挤（那是移动后特性），也不会生成蓝球随从（蓝球的随从是在移动前生成）。

#### 5. 黑球（干扰）
- **移动限制**：黑球**不能主动移动**。  
- **行动方式**：回合内选择**一个敌方弹珠**（必须可见），并指定一个**大致方向**（6个主方向之一）。  
  - 系统随机从以下3个方向中选一个：指定方向、指定方向顺时针偏60°、逆时针偏60°（等概率）。  
  - 系统随机从 {2, 3} 中选择移动格数（等概率）。==这个保留？==  
  - 被指定的敌方弹珠**强制**沿选定的方向移动指定格数（移动过程中正常进行碰撞弹性交换）。  
  - **明确规则**：该移动<font color='red'>与正常移动逻辑相同，会与其他弹珠发生正常的交互。</font>若移动过程中或终点出界，则敌方弹珠死亡。  
  - 该移动不视为“黑球的移动”，因此不会触发黑球的其他特性（它本身无移动特性）。

#### 6. 黄球（死亡增益）
- **移动方式**：与白球相同（方向 + 力度1~5 ±1）。  
- **死亡时触发**：当黄球因任何原因死亡（出界、碰撞致死等）时，**立即手动选择己方一个其他弹珠**（不能是白球或黄球），给予永久增益。  
  - 对**蓝球**：其随从出界不再导致蓝球死亡（但蓝球自身出界依然死亡）。  
  - 对**绿球**：推挤范围从相邻1格变为 **相邻 X 格**（X = 1 + 黄球增益次数）。多个黄球死亡可叠加。  
  - 对**红球**：移动步数上限从4提升至 **4 + 增益次数**（例如2次增益后上限6步）。  
  - 对**黑球**：敌方弹珠被强制移动的格数**固定为3**（不再随机2或3）。  
  - 增益对同一弹珠可多次叠加（由不同黄球死亡提供）。  
  - 若增益目标之后死亡，增益消失（不继承）。

### 六、碰撞与结算顺序

1. 弹珠**逐格推进**移动。
    
2. 每进入一个有弹珠的格点，**两者交换速度矢量**（完全弹性碰撞）：当前弹珠停下，被撞弹珠获得剩余步数继续沿原方向移动。
    
3. 每步结束后立即检查出界，出界则死亡。
    
4. 绿球推挤在移动完全停止后**同时结算**。
